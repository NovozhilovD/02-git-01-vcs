# 02-git-01-vcs
123

Файл .gitignore нужен для того, чтобы Git не добавлял в репозиторий служебные, временные и потенциально чувствительные файлы Terraform.

Правила в .gitignore записываются как шаблоны. Символ * означает любое количество любых символов в имени файла, но не включает символ /. Например, правило *.tfvars означает, что будут проигнорированы все файлы, имя которых заканчивается на .tfvars: terraform.tfvars, dev.tfvars, prod.tfvars и другие похожие файлы.

Правило *.tfstate означает, что Git будет игнорировать файлы состояния Terraform с расширением .tfstate, например terraform.tfstate. Правило *.tfstate.* означает, что будут проигнорированы файлы, у которых после .tfstate. есть любое продолжение, например terraform.tfstate.backup.

Правило crash.log игнорирует файл с точным именем crash.log, а правило crash.*.log игнорирует crash-логи, у которых между crash. и .log есть любой набор символов, например crash.12345.log.

Слеш / в конце правила означает, что правило относится к каталогу. Поэтому правило .terraform/ означает, что Git будет игнорировать каталог .terraform и всё его содержимое. В этом каталоге Terraform хранит загруженные провайдеры и модули.

Две звёздочки ** в .gitignore используются для поиска по вложенным каталогам. Например, шаблон **/example.tf означал бы файл example.tf в любом каталоге, а шаблон logs/** означал бы всё содержимое каталога logs на любой глубине. Конструкция **/ означает любое количество вложенных каталогов. Например, **/.terraform/ означало бы игнорировать каталог .terraform независимо от того, насколько глубоко он расположен.

Также благодаря .gitignore в репозиторий не будут попадать файлы переменных *.tfvars и *.tfvars.json, которые могут содержать пароли, токены, ключи и другие секретные значения. Кроме того, будут игнорироваться локальные override-файлы override.tf, override.tf.json, *_override.tf, *_override.tf.json, временный lock-файл .terraform.tfstate.lock.info, а также локальные CLI-конфигурации .terraformrc и terraform.rc.
